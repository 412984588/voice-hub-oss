## 工作流程

1. **提交阶段**：先执行 git commit 保存已完成的工作
2. **扫描阶段**：使用 `grep -rn "TODO\|FIXME\|XXX" --include="*.ts" --include="*.tsx" .` 扫描所有 TODO
3. **分类阶段**：按优先级对 TODO 分类（高/中/低）
4. **实现阶段**：逐一实现高优先级 TODO，每个实现后运行 `pnpm test` 验证
5. **提交阶段**：每完成一批 TODO，创建一个原子提交

## 执行规则

- 每个 TODO 实现后立即运行 `pnpm test` 验证
- 测试通过后创建原子提交（单次提交 ≤ 50 行）
- 遇到需要设计的 TODO，先规划后实现
- 不属于本轮的 TODO（如"未来优化"）保留原样
